# 情绪处理指南

## 情绪识别信号

### 愤怒信号
- 感叹号多："什么破东西！！！"
- 威胁性语言："投诉""退钱""差评""12315"
- 负面词汇密集："垃圾""骗人""坑人""辣鸡"
- 质疑品牌："你们这个牌子不行""再也不买了"
- 连续短句发消息（情绪急促）

### 失望信号
- "算了""不用了""直接退吧"
- "早知道不买了"
- 语气平淡但明确要退货/退款

### 焦虑信号
- "急""赶时间""什么时候能到""快点"
- 反复追问同一个问题
- 出差/旅行/送礼等时间压力

### 信任信号
- "麻烦你帮我看看""谢谢"
- "你们家品质一直不错"
- "朋友推荐来的""之前买过"

## 情绪处理原则

### 黄金法则：先情绪，后问题
人在情绪中时，听不进解决方案。必须先让情绪落地。

### 三不原则
1. **不争辩**：用户说什么都不反驳
2. **不推脱**：不说"这不是我们的问题"
3. **不催促**：不催用户"您快决定"

### LEAD模型
- **L**isten（倾听）：认真看用户说了什么，不要急着回复
- **E**mpathize（共情）：表达理解
- **A**cknowledge（承认）：承认问题的存在
- **D**o（行动）：立即做一件事

## 高难度场景处理

### 场景1：用户要投诉/打12315
**处理步骤**：
1. "亲，非常抱歉让您有这么大的不满。您的感受我完全理解。"
2. "在您投诉之前，能不能给我一个机会先帮您解决？我保证给您一个满意的结果。"
3. 给出超出常规的解决方案（退款+补偿/直接换新/额外赠品等）
4. 如果用户坚持投诉，不阻拦："如果您觉得需要投诉，这也是您的权利。同时我这边也会继续帮您处理好这个问题的。"

### 场景2：用户说"再也不买了"
**处理步骤**：
1. 不要急着挽留
2. "亲，这次确实是我们没做好，让您失望了。"
3. 专注于解决当前问题
4. 问题解决后："这次的体验确实不好，我已经帮您反馈给产品组了。希望以后还有机会为您服务😊"
5. 让用户自己决定，不强推

### 场景3：用户重复来售后（同一个人多次进线）
**处理步骤**：
1. 先查看之前的记录
2. "亲，我看到您之前也联系过我们，真的非常抱歉让您反复折腾。"
3. 给出更高级别的解决方案，避免用户第三次进线
4. "这次我一定帮您彻底解决好，不让您再跑一趟了。"

### 场景4：用户犹豫退不退货
**处理步骤**：
1. 不要替用户决定
2. "亲，退和不退都可以的，我都帮您处理。如果是因为[问题]想退的话，我先帮您看看能不能解决"
3. 提供解决方案 + 退货方案，让用户自己选
4. 尊重用户的任何选择

## ⭐ 情境叠加处理（情绪 × 人物情境）

> 当用户同时带有情绪状态和特殊生活情境时（比如愤怒的孕妇、焦虑的老年人），需要组合处理。

### 处理优先级

```
当情绪 + 情境同时存在时：
  第一步：永远先处理情绪（情绪不落地，什么方案都听不进去）
  第二步：在方案中自然融入情境关怀
  第三步：收尾时带入情境祝福/提醒
```

### 常见叠加场景

| 叠加组合 | 处理策略 | 话术要点 |
|---------|---------|---------|
| 😡 愤怒 + 🤰 孕妈 | 先3句安抚情绪 → 方案中加上门取件+先发后退 → 收尾祝福 | "您别着急，气坏了身子不值当的～这个问题我全权帮您处理，您在家等就好" |
| 😡 愤怒 + 👶 宝妈 | 先确认孩子安全 → 道歉安抚 → 方案快速到位 | "先看看宝宝有没有伤到哈——壳子的事交给我，我马上给您处理" |
| 😐 不满 + 👴 老人 | 安抚一句 → 超简步骤 → 主动提供图文 | "叔叔/阿姨您别着急，我一步步教您，特别简单的" |
| 😐 不满 + ✈️ 出差 | 安抚 → 时间线管理 → 临时方案 | "确实赶时间，我帮您看最快的方案——" |
| 😔 低谷 + 🎓 学生 | 直接解决一切 → 免运费 → 给温暖 | "交给我就好，运费我们出，您不用操心了😊" |
| 焦虑 + 🎁 送礼 | 理解时间压力 → 给出明确时间承诺 → 备选方案 | "我帮您算了一下时间，今天发出X号能到，赶得上的！" |

### 叠加处理的三个禁忌

1. **不要同时说太多**：情绪安抚和情境关怀各一句就够，不要写成小作文
2. **不要点破情境**：不说"考虑到您怀孕""因为您年纪大"——用行动而非语言
3. **不要因为同情而过度承诺**：关怀≠给超权限的补偿，关怀是在权限内做到最人性化
