# 案例撰写模板与评审标准

## 零、提交前必过的自检关（最重要）

> 写完案例之前，先用这个表格对照。不通过的不要提交。

### Step 1：先判断这个动作算不算价值创造

**以下属于基本职责，不计入评选，直接排除：**
- [ ] 只是在规定时效内处理了问题（时效性）
- [ ] 照搬SOP/话术模板完成了退换货（按标准处理）
- [ ] 帮用户查了物流/催了件（常规物流查询）
- [ ] 用户申请退款，按规则通过了（常规退款）

**如果上面全都勾了，这个case没有写成案例的价值。**

---

### Step 2：判断价值类型（至少符合一种）

**类型一：通过专业知识，正确处理售后**
- [ ] 用户描述模糊，但我准确判断了真实原因（不是让用户猜）
- [ ] 用了SOP之外的产品/规则知识解决了问题
- [ ] 避免了"按SOP走会出错"的情况，主动调整了方案

**类型二：通过售后解释，自然促成复购（无主动销售）**
- [ ] 用户解决问题后主动说要再买/已经下单了
- [ ] 用户主动让我推荐其他商品（不是我先开口）
- [ ] 用户主动分享/推荐给朋友，有截图证明
- ⚠️ 注意：你开口推荐 → 不算；必须是用户主动行为

**类型三：个性化服务（不只是改变方案）**
- [ ] 我识别了用户没说出口的隐藏需求
- [ ] 我做了用户没预期到的额外动作（不是被要求的）
- [ ] 用户明确表达了"你不一样"/"你是第一个帮我想到的"
- [ ] 在合理范围内做了SOP没写的事

**关键判断**：
- ❌ 用户要A，你给了B（仍然是标准选项）→ 不算
- ✅ 用户要A，你给了B + 额外动作 + 用户感受到特殊对待 → 算

---

### Step 3：加分项检查

- [ ] 有用户原话/截图作为结果证据
- [ ] 有可量化数字（挽留金额/挽留率/回购）
- [ ] 动作可复制（别人看了也能学）
- [ ] 没有主动邀请好评/收藏/推荐

---

## 一、好案例的标准

### 核心判断：有没有"不一样"
一个好案例必须回答一个问题：**你做了什么常规SOP不会做的事？**

### 评审加分项
1. ✅ 有明确的"超预期动作"描述
2. ✅ 有用户的正面反应（截图/原话）
3. ✅ 有可量化的结果（挽留退款金额/回购/推荐）
4. ✅ 动作可复制（别人看了也能学着做）
5. ✅ 体现了"以创造者为本"的价值观

### 评审扣分项
1. ❌ 只描述了常规处理流程
2. ❌ 没有用户反馈/结果
3. ❌ "超预期"是主动要好评/邀请收藏
4. ❌ 描述太模糊，看不出具体做了什么
5. ❌ 结果靠猜测不靠证据

---

## 二、STAR-V法则（案例撰写框架）

比标准STAR多了V（Value），专门对应价值创造考核：

| 要素 | 说明 | 占比 |
|------|------|------|
| **S**ituation | 用户的情况和问题 | 15% |
| **T**ask | 常规SOP会怎么做 | 10% |
| **A**ction | 你做了什么不一样的 | 40% |
| **R**esult | 结果怎么样 | 25% |
| **V**alue | 价值创造的意义 | 10% |

### 重点：Action部分
Action是案例的灵魂，必须包含：
1. **具体动作**：我做了什么（不是"给了更好的服务"）
2. **话术原文**：我说了什么（尽量还原）
3. **为什么这样做**：我的思考逻辑

---

## 三、价值类型分类及写法要点

### 1. 挽留退款类
**写法重点**：展示你如何通过沟通让用户从"一定要退"到"不退了"
- 必须写明用户最初的退款诉求
- 必须写明你的关键转折话术
- 必须写明最终结果（退款取消/改为换货）
- 加分：写出挽留的金额

### 2. 回购转化类
**写法重点**：展示用户如何因为好服务而再次购买
- 必须是用户自发回购，不是你推销
- 必须能说清楚回购和你的服务之间的因果关系
- 加分：有回购订单截图

### 3. 口碑传播类
**写法重点**：展示用户主动向他人推荐
- 必须是用户自发推荐，不是你邀请
- 必须有推荐的证据（用户原话/截图）
- 加分：有被推荐人实际购买的数据

### 4. 情感连接类
**写法重点**：展示你如何建立超越交易关系的连接
- 有用户真心感谢的原话
- 展示你做了什么让用户感受到温暖
- 加分：用户后续主动找你/指定找你

### 5. 问题解决类
**写法重点**：展示你如何创造性地解决了SOP没覆盖的问题
- 必须写清楚SOP没有覆盖什么
- 必须写清楚你自己想了什么办法
- 加分：你的方案被纳入新的SOP

## 四、案例范文框架

### 范文1：挽留退款类

**📌 基本信息**
- 日期：2026-04-15
- 产品：GaN 65W充电器
- 问题：无法给华为手机快充，用户要求退款
- 价值类型：挽留退款

**❓ 用户遇到了什么问题？**
用户购买65W GaN充电器后发现无法给华为Mate 60快充，认为产品虚标功率，要求退款退货。

**🔵 常规流程会怎么做？**
确认协议兼容性 → 如不兼容同意退货 → 如兼容让用户换线测试。

**🟢 我做了什么不一样的事？**
1. 没有直接说"兼容/不兼容"，而是先帮用户排查具体原因
2. 发现用户用的是普通USB-A线（不支持快充协议传输），我科普了"快充三件套"概念：充电器+线+手机都要支持
3. 建议用户换一根支持快充的C-C线试试，同时推荐了我们家对应的线
4. 话术："亲，快充其实需要'三件套'都匹配才行——充电器、线和手机。您的充电器和手机都支持快充，但这根线可能是'瓶颈'。您试试换根C口线？"

**✅ 结果怎么样？**
用户换线后成功快充，取消退货，并追加购买了一根快充线（+39元）。
用户原话："原来是线的问题！谢谢你帮我排查，专业👍"

**💡 这个案例说明了什么？**
很多用户的"产品不好用"其实是使用方式问题。用专业知识帮用户排查而不是直接退货，既能留住用户又能增加连带销售。

### 范文2：口碑传播类

**📌 基本信息**
- 日期：2026-04-10
- 产品：MagSafe磁吸充电宝
- 问题：充电宝摔了一下后磁吸变弱
- 价值类型：口碑传播

**❓ 用户遇到了什么问题？**
用户的MagSafe充电宝不小心摔了一下，磁吸力明显变弱。

**🔵 常规流程会怎么做？**
确认是否在保 → 人为损坏不在保修范围 → 建议付费维修或重新购买。

**🟢 我做了什么不一样的事？**
1. 没有一开始就说"人为损坏不保修"
2. 先关心："亲，摔了一下手机没事吧？充电宝扛摔一点😊"
3. 远程排查：让用户拍照发来看磁铁位置有没有移位
4. 发现只是外壳稍微变形导致磁铁距离变远，建议用户轻轻按压壳体
5. 用户按压后恢复正常！省去了退换的麻烦

**✅ 结果怎么样？**
问题当场解决，用户没花一分钱。
两天后用户回来："你上次帮我的那个客服太厉害了，我跟我同事说了，她也下单了一个😊"

**💡 这个案例说明了什么？**
用专业知识帮用户解决小问题，比机械走退换流程更让用户感动。用户感动了就会自发传播。
